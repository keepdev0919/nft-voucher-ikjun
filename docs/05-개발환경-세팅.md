# 개발환경 세팅

## 공통 (모두 설치)

- Node.js 18+
- MetaMask 브라우저 확장 프로그램 설치
- Ganache 로컬 네트워크를 MetaMask에 추가:
  - 네트워크 이름: Ganache
  - RPC URL: http://127.0.0.1:7545
  - 체인 ID: 1337
  - 통화 기호: ETH

---

## 블록체인 팀 (조현, 황종훈)

### 설치
```bash
npm install -g truffle
npm install -g ganache
```

### 매일 개발 시작할 때
```bash
# 터미널 1: Ganache 실행 (--deterministic 필수)
ganache --port 7545 --chain.chainId 1337 --deterministic

# 터미널 2: 컨트랙트 배포
cd blockchain
npm install
truffle migrate --network development
```

### 배포 후 반드시 해야 할 것
- `truffle migrate` 결과에서 `Voucher` 컨트랙트 주소 복사
- 프론트엔드 팀에 주소 전달 (슬랙/카톡)
- `abi.json` 파일 최신화해서 공유

## 백엔드 팀 (정재현)

### 설치
```bash
brew install mysql redis
brew services start mysql
brew services start redis
mysql -u root -e "CREATE DATABASE IF NOT EXISTS nft_voucher;"
mysql -u root -e "CREATE DATABASE IF NOT EXISTS ttocket;"
```

### 매일 개발 시작할 때
```bash
cd backend
./gradlew bootRun
```

> `Table 'user' already exists` 같은 WARN 로그는 정상. DB에 테이블이 이미 있어서 나는 거고 에러 아님.

## 실행 순서 (항상 이 순서로)

| 순서 | 명령어 | 이유 |
|------|--------|------|
| 1 | `ganache --port 7545 --chain.chainId 1337 --deterministic` | 로컬 이더리움 네트워크 실행. `--deterministic` 없으면 매번 새 계정 생성됨 |
| 2 | `cd blockchain && truffle migrate --network development` | 스마트 컨트랙트를 Ganache에 배포. Ganache는 껐다 켜면 초기화되므로 매번 해야 함 |
| 3 | MySQL, Redis는 brew services로 자동 실행 중 | 백엔드가 DB와 캐시에 의존하므로 먼저 떠있어야 함 |
| 4 | `cd backend && ./gradlew bootRun` | Spring Boot 백엔드 실행 (포트 8080) |
| 5 | `cd frontend && npm start` | React 프론트 실행 (포트 3000) |

## 레포 구조 (조익준 개인 설정)

이 프로젝트는 두 개의 레포로 관리한다.

| 폴더 | 연결된 레포 | 용도 |
|------|------------|------|
| `nft-voucher-ikjun/` | `keepdev0919/nft-voucher-ikjun` | 개인 전체 초안 (backend + blockchain + frontend) |
| `nft-voucher/` | `keepdev0919/blockchain-Voucher` | 팀 협업용 (frontend만) |

### 왜 두 개로 나눴나
- 프론트 담당이지만 팀원들이 개발 시작하기 전에 백엔드/블록체인 초안을 미리 짜둠
- 팀 레포에 백엔드/블록체인까지 올리면 팀원들 작업과 충돌날 수 있어서 분리
- 개인 레포는 참고용, 팀 레포는 협업용

### 프론트 작업 후 팀 레포에 올리는 방법
```bash
# 1. nft-voucher-ikjun에서 프론트 작업 완료
# 2. nft-voucher/frontend에 복사
rsync -a --exclude='.git' nft-voucher-ikjun/frontend/ nft-voucher/frontend/

# 3. 팀 레포에 push
cd nft-voucher
git add frontend/
git commit -m "feat: 바우처 목록 화면 구현"
git push origin main
```

## MetaMask 계정 설정 (최초 1회)

Ganache는 실행 시 가상 계정 10개를 생성하고 각각 1000 ETH를 부여한다.  
MetaMask는 이 계정을 자동으로 인식하지 못하므로, **private key를 import해서 연결**해야 한다.  
연결하면 Ganache의 가상 ETH로 가스비 없이 트랜잭션(바우처 생성, 발급 등)을 실행할 수 있다.

### 순서

**1단계 — Ganache 네트워크를 MetaMask에 추가** (최초 1회)

MetaMask → 네트워크 선택 → 네트워크 추가 → 수동 입력:

| 항목 | 값 |
|------|-----|
| 네트워크 이름 | Ganache |
| RPC URL | http://127.0.0.1:7545 |
| 체인 ID | 1337 |
| 통화 기호 | ETH |

**2단계 — Ganache 계정 import** (최초 1회)

MetaMask → 계정 아이콘 → **계정 추가** → **계정 가져오기** → 아래 private key 입력:

```
0x4f3edf983ac636a65a842ce7c78d9aa706d3b113bce9c46f30d7d21715b23b1d
```

import 완료 후 주소: `0x90F8bf6A479f320ead074411a4B0e7944Ea8c9C1` (1000 ETH 보유)

**3단계 — 이후 매번 할 것**

Ganache 실행 시 MetaMask에서 네트워크를 **Ganache**로, 계정을 **import한 계정**으로 선택하면 끝.

> `US$0.00`으로 표시되는 건 정상 — Ganache ETH는 실제 달러 가치가 없어서 그렇게 보이는 것. ETH 잔고는 실제로 있음.

## 자주 나오는 문제

### `zsh: command not found: start`
`npm start`를 `npm` 엔터 후 `start`로 나눠 입력하면 발생. 한 줄로 입력해야 함.
```bash
npm start
```

### Ganache 재시작 후 MetaMask 트랜잭션 오류
Ganache를 껐다 켜면 블록체인이 초기화되는데 MetaMask는 이전 nonce를 기억하고 있어서 충돌남.  
MetaMask → 설정 → 고급 → 계정 활동 지우기(Clear activity) 하면 해결.
