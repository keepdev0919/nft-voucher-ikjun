# CarbonChain — 블록체인 기반 탄소배출권 거래 플랫폼 아이디어

> TTocket(NFT 공연 티켓 플랫폼) 아키텍처를 탄소배출권 도메인으로 확장한 아이디어

---

## 배경 및 동기

- TTocket의 핵심 가치: "위조 불가 소유권 증명 + 영구 기록"
- 탄소배출권 시장의 핵심 문제: **이중 계산(double counting)** — 같은 배출권을 두 기업이 동시에 보유하는 사기
- NFT는 구조적으로 단 하나의 소유자만 가질 수 있어 이중 계산을 원천 차단
- 공적 신뢰가 필요한 도메인이라 블록체인 투명성 논리가 설득력 있음

> 실제로 국제 탄소 인증 기관 Verra 등이 이중 계산 문제로 신뢰도 타격을 받은 사례가 있음

---

## TTocket과의 구조 대응

| TTocket | CarbonChain |
|---------|-------------|
| 공연 주최자 | 정부 / 인증 기관 |
| 공연 등록 | 탄소 감축 프로젝트 등록 |
| 티켓 (NFT) | 탄소배출권 (NFT, 1개 = 1톤 CO2 감축) |
| 티켓 구매자 | 배출 의무 기업 |
| 암표 방지 | 이중 계산 방지 |
| 티켓 취소/소각 | 배출권 소각 (상쇄 처리) |
| 티켓 보관함 | 기업 보유 배출권 포트폴리오 |

---

## 핵심 플로우

```
1. 정부/인증기관이 탄소 감축 프로젝트 등록 (조림, 재생에너지 등)
2. 감축량 인증 완료 후 → 감축량만큼 NFT 배출권 발행
3. 기업이 지갑(MetaMask 등)으로 배출권 구매 (1차 시장)
4. 기업이 실제 배출량 상쇄 시 → 배출권 NFT 소각 (온체인 기록, 복원 불가)
5. 잉여 배출권은 2차 시장에서 다른 기업에 재거래
6. 정부/감독기관이 전체 발행량·거래량·소각량 실시간 모니터링
```

---

## 블록체인을 써야 하는 이유

| 이유 | 설명 |
|------|------|
| **이중 계산 방지** | NFT는 단 하나의 소유자 → 구조적으로 이중 보유 불가 |
| **소각 기록 불변** | 한번 소각된 배출권은 복원 불가 → 감축 실적 조작 차단 |
| **투명한 감사** | 정부·기업·시민 누구나 발행/거래/소각 현황 조회 가능 |
| **국경 없는 거래** | 스마트컨트랙트로 국가 간 거래도 중개자 없이 처리 가능 |

## 아키텍처 (TTocket 기반)

```
[Frontend] React + Web3.js + MetaMask
     ↕
[Backend] Spring Boot + MySQL + Redis
     ↕
[Blockchain] Solidity (ERC-721) + IPFS
```

### 스마트컨트랙트 주요 함수

| Ticket.sol (TTocket) | Credential.sol (CarbonChain) |
|---------------------|------------------------------|
| `createPerform()` | `registerProject()` — 감축 프로젝트 등록 |
| `createTicket()` | `issueCredit()` — 배출권 NFT 발행 |
| 환불 로직 | `burnCredit()` — 배출권 소각 (상쇄 처리) |
| - | `transferCredit()` — 2차 시장 거래 |

### IPFS 메타데이터 예시

```json

  "issued_at": "2026-04-03",
```

## TTocket에서 재사용 가능한 것

- MetaMask 로그인 → 그대로 사용
- ERC-721 NFT 민팅 로직 → `issueCredit()`으로 변형
- IPFS 메타데이터 저장 → 감축 프로젝트 증빙 문서 저장
- QR 코드 → 배출권 소각 처리 시 확인용

## 새로 추가 필요한 것

- `burnCredit()` — 소각 함수 (TTocket에는 없는 개념)
- 2차 거래 마켓플레이스 (기업 간 재거래)
- 정부용 대시보드 (전체 발행/소각 현황)
- 기업용 대시보드 (보유/사용 포트폴리오)
- 발급 기관 whitelist 관리
