# 00 - 시작하기 (풀스택 로컬 셋업 가이드)

> **이 문서의 목적**: 풀스택 (블록체인 + 백엔드 + 프론트)을 본인 맥북에서 처음부터 끝까지 띄우는 방법.
> 각 명령어가 **무슨 의미인지**도 같이 풀어서 설명.

---

## ⚡ TL;DR — `dev.sh` 스크립트로 한 줄 셋업/실행

매번 ganache + 백엔드 + 프론트 따로 띄우기 귀찮으니 `nft-voucher-ikjun/dev.sh` 스크립트로 묶었어.

```bash
cd nft-voucher-ikjun

# 최초 1회만 — 의존성 설치 + DB 생성 + Ganache 영구화 + 컨트랙트 배포 + 주소 자동 갱신
./dev.sh setup

# 그 다음부터 매일 — 풀스택 시작/종료
./dev.sh up        # 전부 백그라운드로 띄움
./dev.sh down      # 전부 종료
./dev.sh status    # 상태 확인
./dev.sh logs backend   # 백엔드 로그 tail (ganache/backend/frontend/all 중 선택)
```

**핵심 트릭**: Ganache가 `--database.dbPath`로 **영구화**되어 있어서 `down → up` 해도 컨트랙트 그대로 살아있음. 컨트랙트 주소 재갱신 불필요.

스크립트 동작 원리 + 트러블슈팅은 아래에서 자세히.

## 큰 그림 (수동으로 띄울 때)

dev.sh 없이 수동으로 띄울 때는 4개 터미널 필요:

```
┌────────────────────────────────────────────────────────┐
│ 터미널 1: Ganache (가짜 이더리움 블록체인, 포트 7545)   │
│ 터미널 2: Truffle migrate (한 번만 — 컨트랙트 배포)    │
│ 터미널 3: Spring Boot 백엔드 (포트 8080)               │
│ 터미널 4: React 프론트 (포트 3000)                     │
└────────────────────────────────────────────────────────┘

MySQL은 백그라운드 서비스로 항상 떠있어야 함.
```

2. 계정 가져오기 (첫 deterministic 계정 = 백엔드 지갑 = 컨트랙트 owner):
   - Private key: `0x4f3edf983ac636a65a842ce7c78d9aa706d3b113bce9c46f30d7d21715b23b1d`
   - 주소: `0x90F8bf6A479f320ead074411a4B0e7944Ea8c9C1`

### 의존성 설치 (각 디렉토리에서 1회)
```bash

cd ../frontend && npm install
```

## 매일 개발 시작 (4단계)

### 터미널 1 — Ganache 띄우기

`dev.sh`가 추천하는 형태 (블록체인 상태 영구화):
```bash
ganache --port 7545 --chain.chainId 1337 --deterministic --database.dbPath ./.ganache-db
```

**명령어 풀이**:
- `ganache`: 로컬에서 **가짜 이더리움 블록체인**을 띄우는 프로그램. 실제 메인넷에 배포 안 하고 본인 컴퓨터에서 테스트 가능
- `--port 7545`: 7545번 포트로 통신 받음. MetaMask가 이 포트로 연결
- `--chain.chainId 1337`: 체인 ID 1337 (실제 메인넷은 1번, Sepolia 11155111 등). MetaMask에서 어떤 네트워크인지 구분하는 라벨
- `--deterministic`: **매번 같은 계정/주소를 생성**. 안 쓰면 Ganache 재시작할 때마다 새 계정 생성돼서 MetaMask에서 import한 계정이 무효화됨
- `--database.dbPath ./.ganache-db`: **블록체인 상태를 파일로 저장**해서 종료 후 다시 띄울 때 이어서. 컨트랙트 재배포 안 해도 됨. (이 옵션 없으면 메모리에만 저장돼서 종료 시 다 사라짐)

→ 출력에 "Available Accounts" 10개랑 각 계정 private key가 보임. 이 터미널은 **그대로 켜둠**.

### 터미널 2 — 컨트랙트 배포

```bash
cd nft-voucher-ikjun/blockchain
truffle migrate --reset --network development
```

**명령어 풀이**:
- `truffle migrate`: `migrations/` 폴더의 스크립트를 실행해서 컨트랙트를 블록체인에 **배포** (= 코드를 블록체인에 박아넣는 행위)
- `--reset`: 이전에 배포한 기록 무시하고 처음부터 다시 배포. Ganache 재시작했으면 항상 `--reset` 필요
- `--network development`: `truffle-config.js`의 `development` 네트워크 설정(`127.0.0.1:7545`)을 사용

**배포 후 해야 할 일** (필수!):
1. 출력에서 `Voucher` 컨트랙트의 `contract address: 0x...` 복사
2. **백엔드** `application.yml` 또는 환경변수에 `blockchain.contract-address` 갱신
3. **프론트** `frontend/.env.local` 파일에 `REACT_APP_CONTRACT_ADDRESS=0x...` 갱신

이 터미널은 한 번 돌리면 끝. 종료해도 됨.

#### 자주 나는 함정 — `approveMerchant` 셋업

컨트랙트 배포 직후 **백엔드 지갑이 `approvedMerchant`로 등록**되어 있어야 결제(`useVoucherByMerchant`)가 동작함. 두 가지 방법:

- **자동화 (권장)**: `migrations/2_deploy_voucher.js` 안에 한 줄 추가
  ```javascript
  const Voucher = artifacts.require("Voucher");
  module.exports = async function (deployer, network, accounts) {
    await deployer.deploy(Voucher);
    const instance = await Voucher.deployed();
    await instance.approveMerchant(accounts[0], true);  // 백엔드 지갑 화이트리스트
  };
  ```
- **수동**: 어드민 로그인 후 `POST /api/members/merchant/{백엔드주소}/approve?approved=true` 호출

### 터미널 3 — 백엔드 띄우기

```bash
cd nft-voucher-ikjun/backend
./gradlew bootRun
```

**명령어 풀이**:
- `./gradlew`: Gradle Wrapper. 프로젝트가 요구하는 정확한 Gradle 버전을 자동으로 다운받아서 사용. 본인 컴퓨터에 Gradle 미리 깔 필요 없음
- `bootRun`: Spring Boot 앱을 띄우는 Gradle 태스크. 코드 컴파일 → 실행까지 자동

**필요한 환경 설정** (`application.yml`):
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/nft_voucher
    username: root
    password: <본인 MySQL 비번>
  jpa:
    hibernate:
      ddl-auto: update    # 엔티티 변경 시 자동으로 테이블 스키마 갱신

jwt:
  secret: <긴 랜덤 문자열, 32자 이상>
  expiration: 86400000     # JWT 유효시간 (ms 단위, 24시간)

blockchain:
  rpc-url: http://127.0.0.1:7545
  contract-address: <truffle migrate 후 출력 주소>
  backend-wallet-private-key: 0x4f3edf983ac636a65a842ce7c78d9aa706d3b113bce9c46f30d7d21715b23b1d
```

→ 백엔드가 정상 떴으면 `http://localhost:8080`에 떠있고, Swagger는 `http://localhost:8080/swagger-ui/index.html`

### 터미널 4 — 프론트 띄우기

```bash
cd nft-voucher-ikjun/frontend
npm start
```

**명령어 풀이**:
- `npm start`: `package.json`의 `scripts.start`에 정의된 명령(보통 `react-scripts start`) 실행. React 개발 서버를 띄움
- 자동으로 브라우저 열고 `http://localhost:3000` 접속

**필요한 환경변수** (`.env.local` 파일, 없으면 생성):
```
REACT_APP_CONTRACT_ADDRESS=0x<truffle migrate 후 출력 주소>
```

## 시연 흐름 — 두 가지 결제 방향

### 새 방향 (메인, 교수님 권유)
1. 가맹점이 `/merchant/payment-request` → 금액 입력 → QR 띄움
2. 사용자가 폰/다른 창에서 `/voucher/pay` → 카메라로 QR 스캔
3. 본인 바우처 선택 → MetaMask 서명 → 결제 완료
4. 가맹점 화면 자동으로 "결제 완료!" 전환

### 옛 방향 (백업)
1. 사용자가 `/voucher/detail/{id}` → 본인 바우처 QR 띄움
2. 가맹점이 `/merchant/scan` → 카메라로 사용자 QR 스캔
3. 가맹점이 금액 입력 → "결제 요청"
4. 사용자 폰의 VoucherHome에 결제 요청 모달 자동 표시 → 서명 → 결제 완료

## 자주 막히는 곳 (트러블슈팅)

### `Cannot find module 'ganache'` (truffle migrate 시)
→ `blockchain/`에서 `npm install` 안 한 거. `cd nft-voucher-ikjun/blockchain && npm install` 실행

### `Could not find suitable configuration file` (truffle 시)
→ `blockchain/` 디렉토리 아닌 곳에서 truffle 실행한 거. `truffle-config.js`가 있는 디렉토리로 이동

### `µWS is not compatible with your Node.js build` (경고)
→ Node v24가 너무 최신. **무시해도 됨** (fallback 동작). 거슬리면 nvm으로 Node 18 LTS 설치

### MetaMask 트랜잭션 오류 ("Nonce too high" 등)
→ Ganache 재시작했는데 MetaMask가 옛 nonce 기억하고 있는 거. **MetaMask → 설정 → 고급 → "계정 활동 지우기"** 누르면 해결

### `Table 'XXX' already exists` WARN 로그 (백엔드)
→ 정상. JPA의 `ddl-auto=update`가 이미 있는 테이블 보고 알림 띄우는 거. 에러 아님

### 401 Unauthorized (프론트에서 API 호출 시)
→ JWT가 없거나 만료. axios 인터셉터가 자동으로 `/login`으로 리다이렉트할 거. 다시 로그인하면 됨

### 카메라 권한 거부 (폰에서 QR 스캔 시)
→ HTTPS 환경 필요. `localhost`는 OK인데 LAN IP(`192.168.x.x`)는 NG.
- 폰에서 시연하려면 ngrok 같은 HTTPS 터널 필요: `ngrok http 3000`
- 또는 맥북에서 창 두 개로 시연 (가맹점 + 사용자)

### `payment-session` 호출 시 401 또는 403
→ 가맹점 JWT로 로그인했는지 확인. ADMIN은 prepare/execute 불가, MERCHANT 역할만 가능

### `useVoucherByMerchant` revert ("unapproved merchant")
→ 컨트랙트 배포 후 `approveMerchant(백엔드주소, true)` 호출 누락. 어드민 API 또는 migration 자동화로 해결

## 매번 새 Ganache 띄울 때 체크리스트

Ganache를 재시작했다면 매번 이거 다 해야 함:

- [ ] Terminal 1에 `ganache --port 7545 --chain.chainId 1337 --deterministic` 띄움
- [ ] Terminal 2에서 `truffle migrate --reset --network development` 다시 배포
- [ ] 새 컨트랙트 주소를 백엔드 `application.yml`에 갱신
- [ ] 새 컨트랙트 주소를 프론트 `.env.local`에 갱신
- [ ] `approveMerchant(백엔드지갑, true)` 호출 (migration 자동화 안 했으면)
- [ ] MetaMask → "계정 활동 지우기" (옛 nonce 캐시 제거)
- [ ] 백엔드 재시작 (`./gradlew bootRun`)
- [ ] 프론트 재시작 (`npm start`)
