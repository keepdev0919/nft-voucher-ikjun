# UX 플로우

## 등장인물

| 역할 | 설명 | 예시 |
|------|------|------|
| 기관 | 바우처를 만들고 발급하는 곳 | 정부, 복지관, 기업 |
| 사용자 | 바우처를 받아서 쓰는 사람 | 지원금 수혜자, 임직원 |
| 가맹점 | 바우처를 받아주는 가게 | 음식점, 마트, 약국 |

---

## 플로우 1: 기관이 바우처 프로그램 만들기

```
[기관]
  1. MetaMask로 로그인
  2. "바우처 프로그램 생성" 클릭
  3. 정보 입력:
       - 프로그램 이름: "2026 청년 식비 지원"
       - 바우처 금액: 50,000원
       - 총 발행 수량: 1,000개
       - 유효기간: 2026-12-31
       - 사용 가능 카테고리: 음식점, 마트
  4. MetaMask 서명 → 스마트 컨트랙트에 프로그램 등록
  5. 완료 → 대상자 지갑 주소로 바우처 발급
```

**블록체인 동작**: `createVoucherProgram()` 호출 → 체인에 영구 기록

---

## 플로우 2: 사용자가 바우처 받기

```
[사용자]
  1. MetaMask로 로그인
  2. 신청 가능한 바우처 목록 확인
  3. "청년 식비 지원 바우처" 신청 클릭
  4. MetaMask 서명
  5. 내 지갑에 NFT 바우처 민팅됨
  6. "내 바우처" 목록에서 확인
       - 바우처 이름
       - 잔액: 50,000원
       - 유효기간: 2026-12-31
       - QR 코드
```

**블록체인 동작**: `mintVoucher()` 호출 → NFT가 사용자 지갑으로 전송됨

---

## 플로우 3: 가맹점에서 사용하기

```
[사용자]
  1. 가맹점에서 "바우처 사용" 선택
  2. 내 바우처 QR 코드 제시

[가맹점]
  3. QR 스캔
  4. 화면에 표시:
       - 바우처 소유자: 홍길동
       - 바우처 잔액: 50,000원
       - 유효 여부: 유효
  5. 결제 금액 입력: 35,000원
  6. 결제 확인 → 스마트 컨트랙트 실행

[블록체인]
  7. 바우처 사용 처리 (잔액 차감 or 소각)
  8. 사용 기록 영구 저장:
       - 사용 일시, 가맹점명, 결제 금액

[사용자]
  9. 바우처 잔액: 15,000원 (부분 사용 시)
     or 사용완료 → 보관함으로 이동
```

**블록체인 동작**: `useVoucher()` 호출 → 위변조 불가 사용 기록

---

## 플로우 4: 사용 완료 후 보관

```
[사용자]
  - 보관함에서 사용 완료된 바우처 확인
  - "2026-05-01 홍길동 식당 35,000원 사용" 기록
  - 블록체인에 영구 보존
```

## 화면 목록

### 기관 화면
- 로그인
- 바우처 프로그램 생성
- 발급 대상자 관리 (지갑 주소 입력)
- 사용 현황 대시보드

### 사용자 화면
- 로그인
- 신청 가능 바우처 목록
- 바우처 신청
- 내 바우처 목록
- 바우처 상세 + QR 코드
- 사용 내역
- 보관함 (사용완료/만료)

### 가맹점 화면
- 로그인
- QR 스캔
- 결제 금액 입력 및 확인
- 정산 내역

## TTocket과 비교

| TTocket | NFT 바우처 | 변경점 |
|---------|-----------|--------|
| 공연 주최자 | 기관 | 역할 동일, 이름만 다름 |
| 공연 | 바우처 프로그램 | 좌석 개념 없음 |
| 티켓 구매 | 바우처 신청 | 결제 없이 발급 |
| QR 입장 | QR 결제 | 입장 확인 → 금액 차감 |
| 보관함 | 보관함 | 거의 동일 |
| (없음) | 가맹점 | 신규 개념 |
| 좌석 선택 | (없음) | 제거 |
| 취소/환불 | (없음) | 제거 |
